Overview

Page Element Hider is a Firefox extension that allows you to selectively hide unwanted elements on webpages, creating a cleaner and more focused browsing experience. With this tool, you can:

  • Hide distracting elements with a simple right-click
  • Set up automatic hiding rules for specific websites
  • Choose different methods for hiding elements based on your needs
  • Customize how elements are selected and displayed

Basic Usage

Hiding Elements

  1. Navigate to any webpage
  2. Right-click on the element you want to hide
  3. Select "Hide Element" from the context menu
  4. The element will be hidden according to your settings

If you have "Show element border on selection" enabled in the settings, elements will be highlighted when you right-click on them, making it easier to identify what you're about to hide.

Restoring Hidden Elements

To temporarily restore all hidden elements on the current page:

  1. Click on the Page Element Hider icon in your browser toolbar
  2. Select "Show all" in the popup

This will restore all hidden elements for your current browsing session. If you refresh the page, any auto-hide rules will be applied again.

Understanding Hide Types

Page Element Hider offers three different methods for hiding elements, each with its own use cases:

Hide Completely

This option uses display: none to completely remove the element from the page layout. The space occupied by the element will collapse, and other elements will adjust to fill the space.

Best for: Removing advertisements, sidebars, or other elements when you want to reclaim the space they occupy.

Hide (Conceal)

This option uses visibility: hidden to make the element invisible while preserving its space in the layout. The element will be invisible, but the space it occupies will remain empty.

Best for: Hiding elements when you want to maintain the overall page layout without disruption.

Remove

This option actually removes the element from the DOM. This is the most thorough method but can potentially break page functionality if the hidden element is referenced by scripts.

Best for: Completely eliminating elements that might be tracking you or running unwanted scripts.

Use the "Remove" option with caution, as it can sometimes break website functionality if the removed element contains important scripts or is referenced by other parts of the page.

Setting Up Auto-Hide Rules

Auto-hide rules let you automatically hide specific elements whenever you visit a particular website.

Creating Auto-Hide Rules

  1. Visit the website where you want to set up auto-hide rules
  2. Click on the Page Element Hider icon in your browser toolbar
  3. Select "Set auto hide"
  4. In the settings page that opens, you can:
    • Verify the site URL is correct
    • Choose a hide type for this site (or use your default setting)
    • Add CSS selectors for elements you want to hide
  5. Click "Save" to apply your rules

Adding CSS Selectors

CSS selectors tell the extension which elements to hide. You can use:

  • #id - To target an element with a specific ID
  • .class - To target elements with a specific class
  • tag - To target all elements of a specific HTML tag
  • More complex selectors like .class1 .class2 or div > p

If you're not familiar with CSS selectors, you can use browser developer tools to find the right selector. Right-click on an element, select "Inspect" or "Inspect Element", and look for the ID or class attributes in the highlighted HTML.

Managing Auto-Hide Rules

To manage your existing auto-hide rules:

  1. Click on the Page Element Hider icon
  2. Select "Options"
  3. Scroll down to the "Auto-Hide Settings" section
  4. Here you can edit or delete existing rules

Customizing Settings

General Settings

  • Hide Type: Set the default method for hiding elements (Hide completely, Hide/conceal, or Remove)
  • Show element border on selection: When enabled, displays a border around elements when you right-click on them
  • Border settings: Customize the color, style, and width of the selection border

Accessing Settings

To access the settings page:

  1. Click on the Page Element Hider icon in your browser toolbar
  2. Select "Options"

Alternatively, you can access the settings from Firefox's Add-ons Manager:

Troubleshooting

Elements Won't Stay Hidden

If elements you've hidden keep reappearing:

  • The website might be dynamic and regenerate elements after loading
  • Try using a more specific CSS selector in the auto-hide rules
  • Consider using the "Remove" hide type for more persistent elements

Website Functionality Breaks

If a website stops working correctly after hiding elements:

  • Click the extension icon and select "Show all" to restore hidden elements
  • Try using "Hide (conceal)" instead of "Remove" for that particular site
  • Be more selective about which elements you hide - avoid hiding navigation menus or interactive elements

Extension Not Working

If the extension seems to stop working entirely:

  • Make sure the extension is enabled in your Firefox Add-ons Manager
  • Try restarting Firefox
  • If problems persist, try resetting all settings (in Options page) or reinstalling the extension

Frequently Asked Questions

Will my hidden elements stay hidden when I close and reopen Firefox?

Elements that are hidden using auto-hide rules will remain hidden when you reopen Firefox. Elements that were hidden manually with right-click will only stay hidden until you close the tab or refresh the page, unless you set up an auto-hide rule for them.

Can I export my settings to another computer?

Currently, there's no built-in export/import feature. Your settings are stored in Firefox's local storage for the extension.

Does this extension work on all websites?

Page Element Hider works on most websites, but some sites with complex structures or heavy JavaScript might behave unexpectedly. Sites with strict Content Security Policies might also limit some functionality.

Will hiding elements speed up page loading?

Using "Hide completely" or "Hide (conceal)" won't significantly impact page loading as the elements are still downloaded. Using "Remove" might provide some performance benefit if you're removing large elements or elements that run resource-intensive scripts.
